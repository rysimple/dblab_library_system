## 一、设计部分
### 1. 需求分析

#### 1.1 系统概述
图书馆信息管理系统是一个面向高校图书馆的数字化管理平台，旨在实现图书借阅、预约、归还、逾期管理等核心业务的自动化和规范化，支持学生和管理员两种角色。

#### 1.2 用户角色分析

| 角色 | 描述 | 主要功能 |
|------|------|----------|
| 学生 | 图书馆的服务对象 | 查询图书、借书、还书、预约图书、查看借阅记录、查看逾期信息、修改密码、上传头像 |
| 管理员 | 图书馆工作人员 | 管理学生信息、管理图书、查看所有借阅记录、查看所有预约记录、处理逾期记录 |

#### 1.3 功能需求

#####  学生功能模块
- 图书查询：查看所有图书的基本信息（书名、作者、出版社、价格、库存、借阅次数、预约人数），还希望知道图书能否被借出
- 借书：每人最多同时借3本，借期30天，同一天不能重复借阅同一本书，已被预约的图书只有预约者才能借。
- 还书：归还已借图书，逾期自动记录
- 预约图书：图书可被预约，预约有效期7天，同时我们需要知道预约时间、记录预约者和相关书籍
- 查看本人借阅记录
- 查看本人逾期记录
- 个人信息管理：修改本人密码、上传/删除本人头像

##### 管理员功能模块
- 学生管理：查看、编辑、删除学生信息（学号电话密码等）
- 查看图书信息
- 查看管理员信息
- 查看所有学生的借阅记录
- 查看所有有效预约
- 逾期记录管理：查看所有逾期记录，标记已处理，删除记录

#### 1.4 数据需求
通关上面的用户需求分析我们不难考虑设计以下实体：
| 实体 | 属性 | 说明 |
|------|------|------|
| 学生(Student) | `sno`, sname, spassword, sclass, sphone, semail, photo | 存储学生基本信息（学号、姓名、密码、班级、电话、邮箱、头像路径） |
| 管理员(Admin) |  `ano`, aname, apassword, aphone | 存储管理员信息 （管理员姓名、密码、电话） |

| 借阅(Borrow) | `borrow_id`, sno, bid, borrow_date,  return_date | 实现借阅功能 ，需要记录借书人、图书、借阅日期、实际归还日期等基本借阅信息（我们还希望看到实际应还日期，但这个可以由 borrow_date 计算得出） |
| 预约(Reserve) | `reserve_id`, sno, bid, reserve_date, take_date | 记录图书预约，与借阅信息类似 |
| 逾期(Overdue) | `overdue_id`, borrow_id, is_paid, paid_date | 记录逾期处理状态 ，在处理逾期记录时，我们实际还希望显示逾期天数和罚款金额等，但这些可以通过查询`borrow_id`计算得出，依赖 `borrow_id` ，为了满足`3NF`我们不存储他们 |

##### GetCurrentBorrowCount - 获取当前借阅数量

```sql
DELIMITER $$
DROP FUNCTION IF EXISTS GetCurrentBorrowCount$$
CREATE FUNCTION GetCurrentBorrowCount(p_sno VARCHAR(20))
RETURNS INT
DETERMINISTIC
BEGIN
    DECLARE cnt INT;
    SELECT COUNT(*) INTO cnt FROM Borrow 
    WHERE sno = p_sno AND return_date IS NULL;
    RETURN cnt;
END$$
DELIMITER ;
```

**功能说明**：输入学生学号，返回该学生当前未归还的借阅数量。在借书时检查是否达到3本上限，在管理员查看学生信息时显示当前借阅数。

#### 2.3 存储过程

系统共设计 3个存储过程，分别处理借书、还书和删除逾期记录业务。

##### 存储过程1：BorrowBook - 借书

```sql
-- 8. 存储过程 - 借书
DROP PROCEDURE IF EXISTS BorrowBook;
DELIMITER $$

CREATE PROCEDURE BorrowBook(
    IN p_sno VARCHAR(20),      -- 输入：学生学号
    IN p_bid VARCHAR(20),      -- 输入：图书编号
    OUT p_msg VARCHAR(100),    -- 输出：提示信息
    OUT p_code INT             -- 输出：状态码（0成功，其他失败）
)
BEGIN
    DECLARE v_available INT;           -- 图书可用库存
    DECLARE v_borrow_count INT;        -- 学生当前借阅数量
    DECLARE v_reserve_exists INT;      -- 该学生是否已预约
    DECLARE v_today_borrow INT;        -- 当天是否借过此书
    DECLARE v_other_reserve INT;       -- 是否有其他人预约
    
    -- 异常处理：发生错误时回滚事务
    DECLARE EXIT HANDLER FOR SQLEXCEPTION
    BEGIN
        ROLLBACK;
        SET p_msg = '系统错误，借书失败';
        SET p_code = 99;
    END;
    
    START TRANSACTION;
    
    -- ① 检查当天是否重复借阅
    SELECT COUNT(*) INTO v_today_borrow
    FROM Borrow 
    WHERE sno = p_sno AND bid = p_bid AND borrow_date = CURDATE();
    
    IF v_today_borrow > 0 THEN
        SET p_msg = CONCAT(p_bid, '：借书失败：同一天不允许重复借阅');
        SET p_code = 4;
        ROLLBACK;
    ELSE
        -- ② 检查借阅数量（上限3本）
        SET v_borrow_count = GetCurrentBorrowCount(p_sno);
        IF v_borrow_count >= 3 THEN
            SET p_msg = CONCAT(p_bid, '：借书失败：已达借阅上限3本');
            SET p_code = 1;
            ROLLBACK;
        ELSE
            -- ③ 检查库存
            SELECT available_count INTO v_available FROM Book WHERE bid = p_bid;
            
            IF v_available IS NULL THEN
                SET p_msg = CONCAT(p_bid, '：借书失败：图书不存在');
                SET p_code = 3;
                ROLLBACK;
            ELSEIF v_available <= 0 THEN
                SET p_msg = CONCAT(p_bid, '：借书失败：图书已无库存');
                SET p_code = 2;
                ROLLBACK;
            ELSE
                -- ④ 检查预约情况
                -- 检查该学生是否已预约
                SELECT COUNT(*) INTO v_reserve_exists 
                FROM Reserve 
                WHERE bid = p_bid AND sno = p_sno 
                  AND take_date IS NULL 
                  AND DATE_ADD(reserve_date, INTERVAL 7 DAY) >= CURDATE();
                
                -- 检查是否有其他学生预约
                SELECT COUNT(*) INTO v_other_reserve 
                FROM Reserve 
                WHERE bid = p_bid AND sno != p_sno 
                  AND take_date IS NULL 
                  AND DATE_ADD(reserve_date, INTERVAL 7 DAY) >= CURDATE();
                
                -- 若有其他人预约且当前学生未预约，则不允许借
                IF v_other_reserve > 0 AND v_reserve_exists = 0 THEN
                    SET p_msg = CONCAT(p_bid, '：借书失败：图书已被预约，只有预约者能借');
                    SET p_code = 5;
                    ROLLBACK;
                ELSE
                    -- ⑤ 插入借阅记录
                    INSERT INTO Borrow(sno, bid, borrow_date, return_date)
                    VALUES(p_sno, p_bid, CURDATE(), NULL);
                    
                    -- 若该学生有预约，借书成功后自动删除预约记录
                    IF v_reserve_exists > 0 THEN
                        DELETE FROM Reserve 
                        WHERE bid = p_bid AND sno = p_sno 
                          AND take_date IS NULL 
                          AND DATE_ADD(reserve_date, INTERVAL 7 DAY) >= CURDATE();
                    END IF;
                    
                    SET p_msg = CONCAT(p_bid, '：借书成功');
                    SET p_code = 0;
                    COMMIT;
                END IF;
            END IF;
        END IF;
    END IF;
END$$

DELIMITER ;
```

| 检查项 | 规则 | 错误码 |
|--------|------|--------|
| 重复借阅 | 同一天不能重复借同一本书 | 4 |
| 借阅上限 | 每人最多同时借3本 | 1 |
| 库存检查 | 图书available_count必须大于0 | 2 |
| 图书存在 | 图书编号必须存在 | 3 |
| 预约优先 | 被预约的图书只有预约者能借 | 5 |
| 成功 | 借书成功 | 0 |

##### 存储过程2：ReturnBook - 还书

```sql
-- 9. 存储过程 - 还书（动态计算应还日期）
DROP PROCEDURE IF EXISTS ReturnBook;
DELIMITER $$

CREATE PROCEDURE ReturnBook(
    IN p_sno VARCHAR(20),      -- 输入：学生学号
    IN p_bid VARCHAR(20),      -- 输入：图书编号
    OUT p_msg VARCHAR(100),    -- 输出：提示信息
    OUT p_code INT             -- 输出：状态码（0成功，其他失败）
)
BEGIN
    DECLARE v_borrow_id INT;       -- 借阅记录编号
    DECLARE v_borrow_date DATE;    -- 借书日期
    DECLARE v_due_date DATE;       -- 应还日期
    DECLARE v_overdue INT;         -- 逾期天数
    
    -- 异常处理
    DECLARE EXIT HANDLER FOR SQLEXCEPTION
    BEGIN
        ROLLBACK;
        SET p_msg = '系统错误，还书失败';
        SET p_code = 99;
    END;
    
    START TRANSACTION;
    
    -- ① 查询未归还的借阅记录
    SELECT borrow_id, borrow_date INTO v_borrow_id, v_borrow_date
    FROM Borrow 
    WHERE sno = p_sno AND bid = p_bid AND return_date IS NULL
    LIMIT 1;
    
    -- ② 若不存在借阅记录，返回错误
    IF v_borrow_id IS NULL THEN
        SET p_msg = CONCAT(p_bid, '：还书失败：未找到借阅记录');
        SET p_code = 1;
        ROLLBACK;
    ELSE
        -- ③ 计算应还日期 = 借阅日期 + 30天
        SET v_due_date = DATE_ADD(v_borrow_date, INTERVAL 30 DAY);
        
        -- ④ 更新还书日期
        UPDATE Borrow 
        SET return_date = CURDATE()
        WHERE borrow_id = v_borrow_id;
        
        -- ⑤ 计算逾期天数（当前日期 - 应还日期，正数表示逾期）
        SET v_overdue = DATEDIFF(CURDATE(), v_due_date);
        
        -- ⑥ 如果逾期，在Overdue表中插入记录
        IF v_overdue > 0 THEN
            INSERT INTO Overdue(borrow_id, is_paid, paid_date)
            VALUES(v_borrow_id, FALSE, NULL);
            SET p_msg = CONCAT(p_bid, '：还书成功，逾期', v_overdue, '天');
        ELSE
            SET p_msg = CONCAT(p_bid, '：还书成功');
        END IF;
        
        SET p_code = 0;
        COMMIT;
    END IF;
END$$
DELIMITER ;
```

| 规则 | 说明 |
|------|------|
| 借阅期限 | 30天（从借书日起算） |
| 逾期处理 | 自动在Overdue表生成未处理记录 |
| 库存更新 | 由触发器自动完成 |
| 动态计算 | 应还日期每次查询时计算，不存储冗余字段 |

#### 2.4 触发器

系统共设计 5个触发器，用于自动维护数据一致性。

| 触发器名称 | 触发事件 | 操作表 | 执行动作 |
|------------|----------|--------|----------|
| reserve_insert_trigger | INSERT Reserve | Book | reserve_Times +1 |
| reserve_delete_trigger | DELETE Reserve | Book | reserve_Times -1 |
| borrow_insert_trigger | INSERT Borrow | Book | available_count -1，borrow_Times +1 |
| borrow_update_trigger | UPDATE Borrow | Book | available_count +1（还书时） |
##### 触发器1：reserve_insert_trigger - 预约时增加预约人数

```sql
-- 11. 触发器 - 预约插入（增加预约人数）
DROP TRIGGER IF EXISTS reserve_insert_trigger;
DELIMITER $$

CREATE TRIGGER reserve_insert_trigger
AFTER INSERT ON Reserve
FOR EACH ROW
BEGIN
    UPDATE Book SET reserve_Times = reserve_Times + 1 WHERE bid = NEW.bid;
END$$

DELIMITER ;
```

**作用**：向Reserve表插入一条预约记录后，将对应图书的`reserve_Times`字段加1。自动维护图书的预约人数，无需在业务代码中手动更新

##### 触发器2：reserve_delete_trigger - 取消预约时减少预约人数

```sql
-- 12. 触发器 - 预约删除（减少预约人数）
DROP TRIGGER IF EXISTS reserve_delete_trigger;
DELIMITER $$

CREATE TRIGGER reserve_delete_trigger
AFTER DELETE ON Reserve
FOR EACH ROW
BEGIN
    UPDATE Book SET reserve_Times = reserve_Times - 1 
    WHERE bid = OLD.bid AND reserve_Times > 0;
END$$

DELIMITER ;
```

**作用**：从Reserve表删除一条预约记录后，将对应图书的`reserve_Times`字段减1（确保不小于0），自动维护图书的预约人数

##### 触发器3：borrow_insert_trigger - 借书时减少库存、增加借阅次数

```sql
-- 13. 触发器 - 借阅插入（减少库存，增加借阅次数）
DROP TRIGGER IF EXISTS borrow_insert_trigger;
DELIMITER $$

CREATE TRIGGER borrow_insert_trigger
AFTER INSERT ON Borrow
FOR EACH ROW
BEGIN
    IF NEW.return_date IS NULL THEN
        UPDATE Book SET 
            available_count = available_count - 1,
            borrow_Times = borrow_Times + 1
        WHERE bid = NEW.bid;
    END IF;
END$$

DELIMITER ;
```


**作用**：向Borrow表插入一条借阅记录后，`available_count`减1（库存减少），`borrow_Times`加1（总借阅次数增加）。自动维护图书的库存和借阅统计

##### 触发器4：borrow_update_trigger - 还书时增加库存

```sql
-- 14. 触发器 - 借阅更新（还书时增加库存）
DROP TRIGGER IF EXISTS borrow_update_trigger;
DELIMITER $$

CREATE TRIGGER borrow_update_trigger
AFTER UPDATE ON Borrow
FOR EACH ROW
BEGIN
    IF OLD.return_date IS NULL AND NEW.return_date IS NOT NULL THEN
        UPDATE Book SET available_count = available_count + 1
        WHERE bid = NEW.bid;
    END IF;
END$$

DELIMITER ;
```

**作用**：更新Borrow表的return_date字段（即还书操作）将对应图书的`available_count`加1（库存增加）。自动恢复图书库存

##### 2.5 触发器5 event_delete_expired_reserves - 删除过期预约

```sql
DROP EVENT IF EXISTS event_delete_expired_reserves;
DELIMITER $$

CREATE EVENT event_delete_expired_reserves
ON SCHEDULE EVERY 1 DAY
STARTS CONCAT(CURDATE(), ' 19:20:00')
DO
BEGIN

    WHERE take_date IS NULL
```

#### 3.1 数据库连接封装 (models.py)
```python
import pymysql
from config import Config
from contextlib import contextmanager

class Database:
    def __init__(self):
        self.host = Config.MYSQL_HOST
        self.port = Config.MYSQL_PORT
        self.user = Config.MYSQL_USER
        self.password = Config.MYSQL_PASSWORD
        self.database = Config.MYSQL_DATABASE
    
    @contextmanager
    def get_connection(self):
        """获取数据库连接（自动关闭）"""
        conn = pymysql.connect(
            host=self.host,
            port=self.port,
            user=self.user,
            password=self.password,
            database=self.database,
            charset='utf8mb4',
            cursorclass=pymysql.cursors.DictCursor
        )
        try:
            yield conn
        finally:
            conn.close()
    
    def query(self, sql, params=None):
        """执行查询，返回所有结果"""
        with self.get_connection() as conn:
            with conn.cursor() as cursor:
                cursor.execute(sql, params)
                return cursor.fetchall()
    
    def query_one(self, sql, params=None):
        """执行查询，返回一条结果"""
        with self.get_connection() as conn:
            with conn.cursor() as cursor:
                cursor.execute(sql, params)
                return cursor.fetchone()
    
    def execute(self, sql, params=None):
        """执行更新（INSERT/UPDATE/DELETE）"""
        with self.get_connection() as conn:
            with conn.cursor() as cursor:
                affected = cursor.execute(sql, params)
                conn.commit()
                return affected
    
    def call_procedure(self, proc_name, args):
        """调用存储过程，返回输出参数"""
        with self.get_connection() as conn:
            with conn.cursor() as cursor:
                cursor.callproc(proc_name, args)
                conn.commit()
                # 获取输出参数（假设最多3个输出参数）
                cursor.execute(f'SELECT @_{proc_name}_1, @_{proc_name}_2, @_{proc_name}_3')
                result = cursor.fetchone()
                return result
# 创建全局实例
db = Database()
```
该模块负责封装数据库连接操作，提供统一的数据库访问接口，简化业务逻辑层的数据交互:
| 方法 | 参数 | 返回值 | 用途 |
|------|------|--------|------|
| `query(sql, params)` | SQL语句，参数元组 | 所有结果行（列表） | 执行查询，返回多条记录 |
| `query_one(sql, params)` | SQL语句，参数元组 | 单条结果（字典）或None | 执行查询，返回单条记录 |
| `execute(sql, params)` | SQL语句，参数元组 | 受影响行数（int） | 执行INSERT/UPDATE/DELETE |
| `call_procedure(proc_name, args)` | 存储过程名，参数元组 | 输出参数字典 | 调用MySQL存储过程 |

```python
                                  books=books, msg="请选择图书", success=False)
        try:
            # 检查是否已预约
            if db.query_one("SELECT * FROM Reserve WHERE sno=%s AND bid=%s AND take_date IS NULL AND DATE_ADD(reserve_date, INTERVAL 7 DAY)>=CURDATE()", 
                           (session['sno'], bid)):
                return render_template('reserve.html', reserves=get_student_reserves(session['sno']), 
                                      books=books, msg="预约失败：您已预约过该书", success=False)
            
            db.execute("INSERT INTO Reserve(sno,bid,reserve_date,take_date) VALUES(%s,%s,CURDATE(),NULL)", 
                      (session['sno'], bid))
            
            # 刷新图书列表（保持搜索状态）
            books = db.query("SELECT * FROM Book WHERE bname LIKE %s ORDER BY bname", (f'%{search}%',)) if search else get_all_books()
            return render_template('reserve.html', reserves=get_student_reserves(session['sno']), 
                                  books=books, msg="预约成功", success=True)
        except Exception as e:
            return render_template('reserve.html', reserves=get_student_reserves(session['sno']), 
                                  books=books, msg=f"预约失败：{str(e)}", success=False)
    return render_template('reserve.html', reserves=get_student_reserves(session['sno']), 
                          books=books)
```

预约流程：

- 检查是否已有效预约（7天内未取书）

## 四、总结与展望

### 6.1 已完成功能

| 模块 | 功能 |
|------|------|
| 登录模块 | 学生/管理员登录 | 
| 学生模块 | 借书、还书、预约、查询、修改密码、头像管理 | 
| 管理员模块 | 学生管理、记录查询、逾期处理、预约管理 | 
| 数据库 | 存储过程、触发器、函数、事务 | 
| 图片管理 | 学生头像上传、图书封面显示（相关照片均存在在`/static`目录下） | 

### 6.2 系统亮点

1. `数据库设计规范`：满足3NF，动态计算应还日期，无冗余字段
2. `完整数据库对象`：包含函数、存储过程、触发器、事务
3. `图片管理`：支持学生头像和图书封面管理
4. `权限控制`：学生和管理员功能分离
5. `采用b/s架构，界面较为美观`：消息提示、确认弹窗、搜索功能
